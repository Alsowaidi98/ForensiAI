The dashboard is designed for incident-response demonstrations and capstone defence panels. It balances technical depth (live logs, AI-style analytics) with clear, non-technical views for supervisors and managers.

Live log ingestion
Ingests macOS Unified Logs and Windows security / Sysmon logs into a single view. Suspicious events are highlighted and summarised for quick review.
macOS unified log Windows security Sysmon
AI-assisted risk scoring
Each host receives an AI-style heuristic score based on failed logins, network activity and process behaviour. The Project Manager console shows high-risk machines without exposing raw logs.
Host risk bands Bruteforce signals
Project Manager console
A separate read-only console for supervisors with host ranking, ingest health, analyst audit trail and one-click PDF reports. Perfect for viva and demo sessions.

üõ†Ô∏è How the capstone implementation works

ForensiAI is implemented as a Python / Streamlit dashboard. It reads log files generated by helper scripts, parses and enriches the events, then stores them in a local database used by both the analyst view (app.py) and the Project Manager console (pm_console.py).

Technologies
Python, Streamlit, Pandas, SQLite, macOS logging tools and Windows log exports. The demo is self-contained and designed to run on a lab MacBook.
Security focus
The project focuses on visibility and education rather than production-grade security. It demonstrates log-driven investigation and clear reporting for non-technical stakeholders.
Academic context
Built as an in-house capstone project at Bahrain Polytechnic. The system is used to demonstrate good project management, risk assessment and quality control as much as technical skills.
