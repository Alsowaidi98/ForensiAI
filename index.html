How it works

Under the hood, ForensiAI is a pipeline: background ingestors capture logs, a small state database keeps track of user actions, and the dashboards consume this data in two different ways: detailed for analysts and summarised for management.

End‚Äëto‚Äëend flow

  • 1
    Ingest: macOS ingestor scripts stream Unified Logs and persistence changes into CSV files in the outputs/ folder.
  • 2
    Score: a Python layer parses each event, applies rule‚Äëbased risk scoring and maps events to MITRE ATT&CK tactics.
  • 3
    Visualise: the analyst console shows live tails, Top‚Äë5 suspicious events, timelines and correlation views.
  • 4
    Report: the PM console aggregates results per host, surfaces KPIs, and generates a SOC‚Äëstyle PDF for documentation.

Architecture & threat model

ForensiAI is intentionally simple: a single‚Äëmachine prototype with a clear data flow. The design still follows basic secure‚Äëby‚Äëdesign ideas and uses STRIDE to reason about threats.

High‚Äëlevel architecture
  • macOS ingestor scripts (UnifiedLog + persistence folders)
  • CSV log storage in outputs/
  • SQLite database for settings and audit trail
  • Analyst dashboard (Streamlit app)
  • Project Manager dashboard (separate Streamlit app)
STRIDE‚Äëinspired threat thinking
  • Spoofing: restricts access to local machine; simple login gate for both consoles.
  • Tampering: read‚Äëonly PM view and clear separation between dashboards.
  • Repudiation: user activity log tracks logins and key actions.
  • Information disclosure: data stays on Ahmed‚Äôs machine; no cloud upload.

For Project Managers

The PM console is designed for a non‚Äëtechnical supervisor: it hides low‚Äëlevel log fields and focuses on hosts, risk trends and analyst behaviour. Only the PM can see this view.
