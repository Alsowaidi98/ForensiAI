How to install, run and demo ForensiAI.

This documentation focuses on the macOS implementation of ForensiAI: the analyst dashboard (app.py), the Project Manager console (pm_console.py) and the log ingestors for macOS Unified Log. It is written for supervisors, examiners and technical reviewers.

Note: Windows log ingestion and UI is being developed separately as a companion implementation.

âš¡ Quick start on macOS

A short checklist that shows you understand environment setup, Python tooling and how to run the dashboard locally on a MacBook.

1. System requirements

  • macOS (tested on Apple Silicon MacBook Pro with Sonoma).
  • Python 3.11+ and pip installed.
  • Basic terminal access (Terminal or iTerm2).
macOS unified log Python / Streamlit

2. Clone the repository

From the terminal:

git clone https://github.com/Alsowaidi98/ForensiAI.git
cd ForensiAI

3. Create & activate a virtual environment

Keep dependencies isolated from the rest of the system:

python3 -m venv .venv
source .venv/bin/activate   # on macOS

4. Install Python dependencies

Install the packages required by the dashboard and ingestors:

pip install -r requirements.txt

ðŸ§ª Running the analyst dashboard (app.py)

The analyst view is where live macOS logs are ingested, analysed and visualised. It is implemented using Streamlit with a focus on usability during viva and demo sessions.

1. Start the macOS ingestor

The macOS Unified Log ingestor reads logs via the log command, parses them and stores events in CSV / SQLite so the dashboard can consume them. 

./start_forensiai.sh
# or run the dedicated macOS ingestor script directly
python ingest_macos_unifiedlog.py

In the thesis you can highlight that the ingestor is local only â€“ no logs are sent to the cloud, which aligns with data protection requirements.

2. Launch the Streamlit dashboard

From the project root (with the virtual environment active): 

streamlit run app.py

Streamlit will open the analyst dashboard in the browser (typically http://localhost:8501). This interface shows:

  • Live events table with filters and column selection.
  • Top-5 alerts per host.
  • AI-style risk scoring for each machine.
  • Buttons to generate PDF / CSV reports.

ðŸ“ˆ Project Manager console (pm_console.py)

The PM console is a separate read-only view that reuses the same SQLite state but hides low-level log details. This demonstrates separation of duties and stakeholder-friendly reporting.

How to run the PM console

streamlit run pm_console.py

Once running, the PM console provides:

  • Ingestor health metrics and system status banner.
  • Host ranking by average AI risk.
  • Security hotspots and incident summary for the selected window.
  • Recent analyst activity (audit trail) table.
  • One-click PM PDF report export.
Read-only view Audit logging Supervisor friendly

ðŸ§± Architecture & security notes

Use this section in your thesis to explain the high-level design: how logs flow from the operating system into the dashboard and PM console.

High-level data flow

  • Helper scripts collect logs (macOS Unified Log, later Windows logs).
  • Events are parsed into a clean, tabular format (CSV / SQLite).
  • The analyst dashboard (app.py) reads from this state and applies risk scoring plus visual analytics.
  • The PM console (pm_console.py) reads the same state but only exposes summary metrics and audit logs.
Local storage only Separation of duties Explainable scoring

Data protection & future work

ForensiAI is designed as a teaching and demo tool, but the structure can be extended with stronger security controls:

  • Encrypting the SQLite database at rest.
  • Tokenising sensitive usernames or hostnames.
  • Integrating with external threat-intel APIs (for example, VirusTotal) using per-user API keys stored in environment variables.
